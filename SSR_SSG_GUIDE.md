# 📖 SSR/SSG/Pre-rendering — Полное руководство

## 🎯 В чём проблема вашего сайта

### Сейчас (SPA - Single Page Application)

```html
<!-- Что видит поисковик -->
<!DOCTYPE html>
<html>
<head><title>VFD</title></head>
<body>
  <div id="app"></div>  ← ПУСТО!
  <script src="app.js"></script>
</body>
</html>
```

**Проблема:** Яндекс и Google видят пустую страницу до загрузки JavaScript.

---

## 🔧 Типы рендеринга

### 1️⃣ **CSR (Client-Side Rendering)** — текущий вариант

```
Сервер → Пустой HTML → Браузер загружает Vue → Контент появляется
```

**Плюсы:**
- ✅ Простая разработка
- ✅ Быстрые переходы между страницами
- ✅ Не нужен сервер (GitHub Pages подходит)

**Минусы:**
- ❌ **Плохое SEO** — поисковики видят пустой HTML
- ❌ Медленная первая отрисовка (1-3 сек)

---

### 2️⃣ **SSR (Server-Side Rendering)**

```
Браузер → Запрос → Node.js рендерит Vue в HTML → Готовый контент
```

**Плюсы:**
- ✅ **Отличное SEO** — контент в HTML
- ✅ Мгновенная первая отрисовка
- ✅ Динамический контент (цены, наличие)

**Минусы:**
- ❌ **Нужен Node.js сервер** (постоянно запущен)
- ❌ Сложнее в настройке
- ❌ Сервер может упасть

**Когда использовать:**
- Интернет-магазины с динамическими ценами
- Новостные сайты
- Соцсети

---

### 3️⃣ **SSG (Static Site Generation)**

```
Сборка → Vue рендерит все страницы в HTML → 
Сервер отдаёт готовые файлы
```

**Плюсы:**
- ✅ **Отличное SEO** — контент в HTML
- ✅ **Очень быстро** — статические файлы
- ✅ **Надёжно** — нечему падать
- ✅ Не нужен Node.js процесс

**Минусы:**
- ⚠️ Нужно пересобирать при изменениях
- ⚠️ Не подходит для очень динамического контента

**Когда использовать:**
- ✅ **Ваш случай!**
- Сайты-визитки
- Блоги
- Каталог товаров (если цены меняются редко)

---

### 4️⃣ **Pre-rendering** — упрощённый SSG

```
Сборка → Для некоторых страниц генерируем HTML → 
Остальные остаются SPA
```

**Плюсы:**
- ✅ **Хорошее SEO для главных страниц**
- ✅ **Простая реализация**
- ✅ Быстро собирается

**Минусы:**
- ⚠️ Не все страницы индексируются идеально

**Когда использовать:**
- ✅ **Ваш случай!**
- Когда нужно быстро улучшить SEO
- Когда каталог очень большой

---

## 🚀 Что делаем для вас

### Гибридный подход:

```
Главная страница → Pre-rendered HTML (SEO ✅)
О компании → Pre-rendered HTML (SEO ✅)
Портфолио → Pre-rendered HTML (SEO ✅)
Перегородки → Pre-rendered HTML (SEO ✅)
Каталог → SPA (слишком много товаров)
Карточка товара → SPA (динамически из JSON)
```

### Почему это правильно:

1. **Главные страницы для SEO** — поисковики видят контент
2. **Каталог через SPA** — не нужно генерировать 1000+ страниц
3. **Быстро и просто** — сборка 30 секунд
4. **Не нужен Node.js** — статика на Nginx

## 📦 Использование

### Локальная разработка (без изменений)

```bash
npm run dev
```

### Сборка для продакшена

```bash
npm run build        # Собираем проект
npm run prerender    # Генерируем HTML для главных страниц
npm run sitemap      # Обновляем sitemap.xml
```

### Всё вместе (одной командой)

```bash
npm run deploy       # build + prerender + sitemap
```

## 📊 Что происходит при сборке

### Шаг 1: Vite сборка
```
src/ → dist/
Vue компоненты → JavaScript бандлы
index.html → dist/index.html
```

### Шаг 2: Pre-renderer
```
dist/index.html + HTML контент → 
dist/index.html (с контентом внутри #app)
dist/about/index.html (с контентом)
dist/portfolio/index.html (с контентом)
```

### Результат:

```html
<!-- Было (SPA) -->
<div id="app"></div>

<!-- Стало (Pre-rendered) -->
<div id="app">
  <header>...</header>
  <main>
    <h1>Межкомнатные двери в Челябинске</h1>
    <p>Фирменный салон VFD...</p>
    <!-- ВЕСЬ КОНТЕНТ -->
  </main>
</div>
```

## ⚙️ Работа на Beget VPS

### Nginx конфигурация (уже работает):

```nginx
server {
  listen 80;
  server_name vfd74.ru;
  root /var/www/vfd-door/dist;
  index index.html;
  
  location / {
    try_files $uri $uri/ /index.html;
  }
}
```

### Что происходит:

```
Пользователь → vfd74.ru → 
Nginx → /var/www/dist/index.html → 
Отдаёт готовый HTML с контентом
```

## ❓ Вопросы

### "А если я хочу чтобы каталог тоже индексировался?"

Добавьте в `PAGES`:
```javascript
{ path: '/catalog/innova', file: 'catalog/innova/index.html' }
```

Но тогда нужно генерировать для каждой серии (29 товаров).

### "А если цены изменятся?"

Пересобираете:
```bash
npm run deploy
```
30 секунд — и актуальные цены на сайте.

### "А если товаров станет 1000+?"

Тогда нужно переходить на полноценный SSR с Node.js сервером.
Но для 80 товаров — pre-render идеально.
